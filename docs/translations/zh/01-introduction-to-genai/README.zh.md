## 我们如何得到生成式AI？

尽管近年来关于生成式AI模型的宣传非常高调，但这项技术已经有几十年的历史，最早的研究工作可以追溯到60年代。现在，AI具有人类的认知能力，例如[OpenAI ChatGPT](https://openai.com/chatgpt)或[Bing Chat](https://www.microsoft.com/edge/features/bing-chat?WT.mc_id=academic-105485-koreyst)，它们也使用了GPT模型进行Bing搜索。

回顾一下，AI的第一个原型由打字机聊天机器人组成，依赖于从一组专家中提取的知识库，并在计算机中表示。知识库中的答案是由输入文本中出现的关键字触发的。然而，很快就明显这种使用打字机聊天机器人的方法并不可扩展。

### 一种基于统计的AI方法：机器学习

在90年代，随着将统计方法应用于文本分析，出现了一个转折点。这导致开发了新的算法——被称为机器学习——能够从数据中学习模式，而无需进行明确的编程。这种方法允许机器模拟人类语言理解：统计模型在文本标签对上进行训练，使模型能够使用预定义标签对未知输入文本进行分类，表示消息的意图。

### 神经网络和现代虚拟助手

在更近期，硬件技术的技术进步，能够处理更大量的数据和更复杂的计算，促进了AI领域的研究，导致了先进的机器学习算法的开发——称为神经网络或深度学习算法。

神经网络（尤其是循环神经网络——RNNs）显着增强了自然语言处理能力，使文本的含义以更有意义的方式表示，更好地评估句子中单词的上下文。

这是本世纪初诞生的虚拟助手的技术基础，它非常擅长解释人类语言，识别需求，并执行满足需求的操作——例如回答预定义脚本或使用第三方服务。

### 现代生成式AI

因此，这就是我们今天的生成式AI，它可以看作是深度学习的一个子集。

![AI、ML、DL和生成式AI](./images/AI-diagram.png?WT.mc_id=academic-105485-koreyst)

经过几十年的AI领域研究，一种新的模型架构——称为Transformer——克服了RNN的限制，能够将更长的文本序列作为输入。Transformers基于注意机制，使模型能够给接收到的输入赋予不同的权重，“更多地关注”信息最集中的地方，而不考虑它们在文本序列中的顺序。

大多数最近的生成式AI模型——也称为大型语言模型（LLMs），因为它们使用文本输入和输出——确实基于这种架构。这些模型的有趣之处在于，它们使用来自各种来源（如书籍、文章和网站）的大量未标记数据进行训练，可以适应各种任务，并生成具有语法正确性和创造性的文本。因此，它们不仅极大地增强了机器“理解”输入文本的能力，而且使它们能够生成人类语言的原始响应。

## 大型语言模型如何工作？

在下一章中，我们将探讨不同类型的生成式AI模型，但现在让我们先看看大型语言模型的工作原理，重点关注OpenAI GPT（生成式预训练变压器）模型。

* **分词器，将文本转换为数字**：大型语言模型接收文本作为输入，并生成文本作为输出。但是，由于它们是统计模型，它们使用数字比文本序列更有效。这就是为什么在核心模型使用之前，模型的每个输入都会经过一个分词器处理。标记是文本块——由可变数量的字符组成，因此分词器的主要任务是将输入拆分为标记数组。然后，将每个标记映射到标记索引，该索引是原始文本块的整数编码。

![分词示例](./images/tokenizer-example.png?WT.mc_id=academic-105485-koreyst)

* **预测输出标记**：给定n个标记作为输入（最大n因模型而异），模型能够预测一个标记作为输出。然后，将此标记合并到下一次迭代的输入中，以扩展窗口模式，使用户获得一个（或多个）句子作为答案的更好体验。这就解释了为什么如果你玩过ChatGPT，你可能会注意到它有时看起来像在句子中间停止了。

* **选择过程，概率分布**：模型根据当前文本序列之后出现的概率选择输出标记。这是因为模型预测了所有可能的“下一个标记”的概率分布，根据其训练计算。然而，并不总是从结果分布中选择具有最高概率的标记。在这个选择中添加了一定的随机性，以模拟创造性思维的过程，并且可以使用称为温度的模型参数来调整它。

## 我们的初创公司如何利用大型语言模型？

现在我们对大型语言模型的内部工作原理有了更好的了解，让我们看看它们可以执行的最常见任务的一些实际示例，重点关注我们的业务场景。我们说，大型语言模型的主要功能是“从自然语言文本输入中生成文本”。

但是什么样的文本输入和输出呢？
大型语言模型的输入称为提示，输出称为完成，这个术语指的是模型生成下一个标记以完成当前输入的机制。我们将深入探讨什么是提示以及如何设计它以充分利用我们的模型。但是现在，让我们只是说提示可能包括：

* 一个指定我们期望从模型中获得的输出类型的**指令**。这个指令有时可能会嵌入一些示例或其他数据。

    1. 对文章、书籍、产品评论等的摘要进行总结，同时从非结构化数据中提取见解。
    
    ![摘要示例](./images/summarization-example.png?WT.mc_id=academic-105485-koreyst)

    <br>
    
    2. 对文章、论文、作业等进行创意构思和设计。
    
    ![创意写作示例](./images/creative-writing-example.png?WT.mc_id=academic-105485-koreyst)

    <br>
    
* 一个以对话形式提出的**问题**。

![对话示例](./images/conversation-example.png?WT.mc_id=academic-105485-koreyst)

<br>

* 一段需要完善的**文本**，这隐含了对写作帮助的要求。

![文本完成示例](./images/text-completion-example.png?WT.mc_id=academic-105485-koreyst)

<br>

* 一段**代码**，以及要求解释和文档化它的要求，或者要求生成执行特定任务的代码片段的注释。

![编码示例](./images/coding-example.png?WT.mc_id=academic-105485-koreyst)

<br>

以上示例相当简单，不想详细说明大型语言模型的能力。它们只是想展示使用生成式AI的潜力，特别是在教育背景下，但不仅限于此。

此外，生成式AI模型的输出并不完美，有时模型的创造性会对它起反作用，导致输出是一些单词的组合，人类用户可以将其解释为对现实的误解，或者可能是冒犯性的。生成式AI不是智能的——至少在更全面的智能定义中，包括批判性和创造性推理或情感智能；它不是确定性的，也不是可信的，因为可能会将错误的参考、内容和声明等虚构与正确的信息相结合，并以有说服力和自信的方式呈现。在接下来的课程中，我们将处理所有这些限制，并看看我们可以做些什么来减轻它们。

## 知识检查

关于大型语言模型，哪个说法是正确的？

1. 你每次都会得到完全相同的响应。
2. 它可以完美地完成任务，如加法、生成工作代码等。
3. 尽管使用相同的提示，响应可能会有所不同。它在给你一份文本或代码的初稿方面非常擅长，但你需要逐步改进结果。

答案：3，LLM是非确定性的，响应会有所不同，但你可以通过温度设置控制它的变化。你也不应该期望它完美地完成任务，它是为了为你做繁重的工作，这通常意味着你会得到一次不错的尝试，需要逐步改进它。
