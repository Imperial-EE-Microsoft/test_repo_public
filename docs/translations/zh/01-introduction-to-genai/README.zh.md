## 我们如何得到生成式AI？

尽管近年来生成式AI模型的*炒作*引起了极大的关注，但这项技术已经有数十年的历史，最早的研究工作可以追溯到60年代。现在我们的AI具有了人类的认知能力，比如对话，例如[OpenAI ChatGPT](https://openai.com/chatgpt)或[Bing Chat](https://www.microsoft.com/edge/features/bing-chat?WT.mc_id=academic-105485-koreyst)，后者也使用GPT模型进行Bing搜索对话。

回到过去，最早的AI原型由打字机编写的聊天机器人组成，依赖于从一组专家中提取并表示为计算机的知识库。知识库中的答案是由输入文本中出现的关键字触发的。然而，很快就清楚了，使用打字机编写的聊天机器人这种方法并不可扩展。

### 一种基于统计的AI方法：机器学习

90年代出现了一个转折点，即将统计方法应用于文本分析。这导致了新算法的开发——称为机器学习——它能够从数据中学习模式，而不需要进行显式编程。这种方法允许机器模拟人类语言理解：统计模型基于文本-标签对进行训练，使模型能够使用预定义的标签对未知的输入文本进行分类，代表了消息的意图。

### 神经网络和现代虚拟助手

近年来，硬件的技术进步使得处理更大量的数据和更复杂的计算成为可能，这促进了人工智能领域的研究，推动了先进的机器学习算法的开发——称为神经网络或深度学习算法。

神经网络（特别是循环神经网络——RNN）极大地增强了自然语言处理能力，使得文本的意义可以更有意义地表示，更加重视词在句子中的上下文。

这是新世纪初诞生的虚拟助手的技术，非常擅长解释人类语言，识别需求，并执行满足需求的操作——例如使用预定义的脚本回答问题或使用第三方服务。

### 当下的生成式AI

这就是我们如今的生成式AI，它可以被看作是深度学习的一个子集。

![AI，ML，DL和生成式AI](./images/AI-diagram.png?WT.mc_id=academic-105485-koreyst)

经过几十年的人工智能研究，一种新的模型架构——称为*Transformer*——克服了RNN的局限性，能够获取更长的文本序列作为输入。Transformer基于注意机制，使模型能够对其接收到的输入赋予不同的权重，“更多地关注”最相关信息所集中的地方，而不管它们在文本序列中的顺序如何。

大多数最近的生成式AI模型（也称为大型语言模型（LLMs），因为它们使用文本输入和输出）确实基于这种架构。这些模型——在各种来源的未标记数据（如书籍、文章和网站）上进行训练——的有趣之处在于它们可以适应各种任务，并生成具有语法正确性和创造性的文本。因此，它们不仅极大地增强了机器“理解”输入文本的能力，而且还增强了它们生成人类语言原始响应的能力。

## 大型语言模型是如何工作的？

在接下来的章节中，我们将探讨不同类型的生成式AI模型，但现在让我们先看看大型语言模型是如何工作的，重点关注OpenAI GPT（生成预训练变压器）模型。

* **分词器，文本到数字**：大型语言模型接收文本作为输入，并生成文本作为输出。但是，作为统计模型，它们使用数字比文本序列更有效。这就是为什么模型的每个输入在使用核心模型之前都要经过分词器处理。标记是文本的一部分——由可变数量的字符组成，因此分词器的主要任务是将输入拆分为标记数组。然后，将每个标记映射到标记索引，该索引是原始文本块的整数编码。

![分词示例](./images/tokenizer-example.png?WT.mc_id=academic-105485-koreyst)

* **预测输出标记**：给定n个标记作为输入（最大n因模型而异），模型能够预测一个标记作为输出。然后将此标记合并到下一次迭代的输入中，以扩展窗口模式，从而实现更好的用户体验，以获取一个（或多个）句子作为答案。这就解释了为什么，如果你曾经玩过ChatGPT，你可能会注意到有时它看起来像是在句子的中途停止了。

* **选择过程，概率分布**：模型根据当前文本序列之后出现的概率选择输出标记。这是因为模型基于其训练预测所有可能的“下一个标记”的概率分布。然而，并不总是选择概率最高的标记。在这个结果分布中添加了一定程度的随机性，以使模型以非确定性的方式运作——对于相同的输入，我们不会得到完全相同的输出。这种随机性的程度是通过一个称为温度的模型参数进行调整的，以模拟创造性思维的过程。

## 我们的初创公司如何利用大型语言模型？

现在我们对大型语言模型的内部工作有了更好的理解，让我们看看它们可以执行相当不错的最常见任务的一些实际示例，重点关注我们的业务场景。

我们说大型语言模型的主要功能是*从自然语言中的文本输入开始，从头开始生成文本*。

但是什么样的文本输入和输出呢？
大型语言模型的输入称为提示，输出称为完成，这个术语指的是模型生成下一个标记以完成当前输入的机制。我们将深入研究提示是什么以及如何设计提示以充分利用我们的模型。但是现在，让我们只说提示可能包括：

* 指定我们期望从模型中获得的输出类型的**指令**。这个指令有时可能会嵌入一些示例或其他数据。

    1. 对文章、书籍、产品评论等的摘要进行总结，并从非结构化数据中提取洞见。

    ![摘要示例](./images/summarization-example.png?WT.mc_id=academic-105485-koreyst)

    <br>
    
    2. 创意构思和设计文章、论文、作业等。

    ![创意写作示例](./images/creative-writing-example.png?WT.mc_id=academic-105485-koreyst)

    <br>
    
* 以对话的形式提出的**问题**。

    ![对话示例](./images/conversation-example.png?WT.mc_id=academic-105485-koreyst)

* 需要完成的**文本块**，它隐含地要求写作帮助。

以上示例相当简单，不想成为大型语言模型能力的详尽证明。它们只是想展示使用生成式AI的潜力，特别是但不限于教育环境。

此外，生成式AI模型的输出并不完美，有时模型的创造力可能会适得其反，导致输出是由人类用户解释为现实的神秘组合词，或者可能是冒犯性的。生成式AI并不智能——至少在更全面的智能定义中不智能，包括批判性和创造性推理或情感智能；它不是确定性的，也不是可信的，因为可能会将错误的引用、内容和声明等虚构与正确的信息相结合，并以令人信服和自信的方式呈现。在接下来的课程中，我们将处理所有这些限制，并看看我们可以做些什么来减轻它们。

## 知识检测

大型语言模型的哪些说法是正确的？

1. 每次都会得到完全相同的响应。
2. 它可以完美地执行任务，比如加法，生成工作代码等。
3. 响应可能会因使用相同的提示而有所不同。它也非常擅长为你提供一篇文章或代码的初稿，但你需要逐步改进结果。

答案：3，LLM是非确定性的，响应会有所不同，但是你可以通过一个称为温度的设置控制它的方差。你也不应该期望它完美地执行任务，它在这里是为了为你做重活，这通常意味着你会得到一个不错的第一次尝试，需要逐步改进它。
