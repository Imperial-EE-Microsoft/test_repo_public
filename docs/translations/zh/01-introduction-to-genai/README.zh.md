## 我们如何得到生成式AI？

尽管最近宣布生成式AI模型引起了非凡的热潮，但这项技术已经有几十年的历史了，最早的研究工作可以追溯到60年代。现在，AI具有了人类的认知能力，例如对话，例如[OpenAI ChatGPT](https://openai.com/chatgpt)或[Bing Chat](https://www.microsoft.com/edge/features/bing-chat?WT.mc_id=academic-105485-koreyst)，后者也使用GPT模型进行Bing搜索对话。

退后一步，AI的第一个原型是打字聊天机器人，依赖从一组专家中提取并表示到计算机中的知识库。知识库中的答案是由输入文本中出现的关键字触发的。然而，很快就清楚，这种使用打字聊天机器人的方法不太可扩展。

### AI的统计方法：机器学习

在90年代，出现了一个转折点，应用了一种统计方法来分析文本。这导致了新算法的开发，称为机器学习，能够从数据中学习模式，而无需明确编程。这种方法使机器能够模拟人类语言理解：统计模型在文本-标签配对上进行训练，使模型能够将未知的输入文本分类为表示消息意图的预定义标签。

### 神经网络和现代虚拟助手

在更近期，硬件的技术进化，使其能够处理更大量的数据和更复杂的计算，鼓励了AI领域的研究，导致了高级机器学习算法的开发，称为神经网络或深度学习算法。

神经网络（尤其是循环神经网络-RNN）显着增强了自然语言处理能力，使文本的含义以更有意义的方式表示，评估句子中单词的上下文。

这是新世纪初诞生的虚拟助手的动力，它非常擅长解释人类语言，识别需求，并执行满足需求的操作-例如使用预定义脚本回答或使用第三方服务。

### 现代生成式AI

这就是我们如今看到的生成式AI，可以看作是深度学习的一个子集。

![AI，ML，DL和生成式AI](./images/AI-diagram.png?WT.mc_id=academic-105485-koreyst)

经过几十年的AI研究，一种新的模型架构-称为Transformer-克服了RNN的限制，能够将更长的文本序列作为输入。变压器基于注意机制，使模型能够赋予它接收到的输入不同的权重，“更多地关注”最相关信息所集中的地方，而不考虑它们在文本序列中的顺序。

大多数最近的生成式AI模型-也称为大型语言模型（LLM），因为它们使用文本输入和输出-确实基于这种架构。这些模型-在各种来源的大量未标记数据上进行训练，例如书籍、文章和网站-的有趣之处在于，它们可以适应各种任务，并生成具有语法正确性和创造性的文本。因此，它们不仅极大地增强了机器“理解”输入文本的能力，而且使它们能够生成人类语言的原始响应。

## 大型语言模型是如何工作的？

在接下来的章节中，我们将探讨不同类型的生成式AI模型，但现在让我们先看一下大型语言模型的工作原理，重点介绍OpenAI GPT（生成式预训练变压器）模型。

* **分词器，文本转数字**：大型语言模型接收文本作为输入，并生成文本作为输出。然而，由于它们是统计模型，它们使用数字比文本序列更好。这就是为什么在核心模型使用之前，每个模型的输入都要经过分词器处理。令牌是一段文本-由可变数量的字符组成，因此分词器的主要任务是将输入拆分成令牌数组。然后，将每个令牌映射到令牌索引，该索引是原始文本块的整数编码。

![分词示例](./images/tokenizer-example.png?WT.mc_id=academic-105485-koreyst)

* **预测输出令牌**：给定n个令牌作为输入（最大n因模型而异），模型能够预测一个令牌作为输出。然后，将该令牌并入下一次迭代的输入中，以扩展窗口模式，使用户体验更好，可以得到一个（或多个）句子作为答案。这就解释了为什么，如果你玩过ChatGPT，你可能会注意到有时它看起来像在句子的中间停止了。

* **选择过程，概率分布**：模型根据它出现在当前文本序列之后的概率选择输出令牌。这是因为模型根据其训练预测所有可能的“下一个令牌”的概率分布。然而，并不总是从结果分布中选择具有最高概率的令牌。在这个选择中添加了一定程度的随机性，以使模型以非确定性的方式运行-我们不会为相同的输入得到完全相同的输出。这种随机性的程度是为了模拟创造性思维的过程，可以使用称为温度的模型参数来调整它。

## 我们的初创公司如何利用大型语言模型？

现在，我们对大型语言模型的内部工作原理有了更好的了解，让我们看一些实际的例子，说明它们可以很好地执行的最常见任务，重点关注我们的业务场景。

我们说，大型语言模型的主要功能是*从自然语言的文本输入开始，从头开始生成文本*。

但是，是什么样的文本输入和输出呢？

大型语言模型的输入称为提示，输出称为完成，这个术语是指模型生成下一个令牌以完成当前输入的机制。我们将深入探讨什么是提示以及如何设计提示以充分利用我们的模型。但是现在，让我们只说一下提示可能包括：

* **说明**，指定我们期望从模型中获得的输出类型。这个说明有时可能会嵌入一些示例或其他数据。

    1. 文章、书籍、产品评论等的摘要，以及从非结构化数据中提取见解。
    
    ![摘要示例](./images/summarization-example.png?WT.mc_id=academic-105485-koreyst)

    <br>
    
    2. 文章、论文、作业或更多的创意构思和设计。
    
    ![创意写作示例](./images/creative-writing-example.png?WT.mc_id=academic-105485-koreyst)

    <br>
    
* **问题**，以对话形式与代理人交流提出。

![对话示例](./images/conversation-example.png?WT.mc_id=academic-105485-koreyst)

<br>

* 一段需要完成的**文本**，它隐含地要求写作帮助。

![文本完成示例](./images/text-completion-example.png?WT.mc_id=academic-105485-koreyst)

<br>

* 一段**代码**，连同要求解释和记录它的请求，或者是一个注释，要求生成执行特定任务的代码片段。

![编码示例](./images/coding-example.png?WT.mc_id=academic-105485-koreyst)

<br>

上面的示例相当简单，并且不想成为大型语言模型功能的详尽证明。它们只是想展示使用生成式AI的潜力，特别是在教育背景下，但并不限于此。

此外，生成式AI模型的输出并不完美，有时模型的创造性可能会对它起反作用，导致输出是一组人类用户可以解释为现实的谜语或可能具有冒犯性的单词组合。生成式AI并不聪明-至少在包括批判性和创造性推理或情感智能在内的更全面的智能定义中，它不是确定性的，也不是可信的，因为可能会将虚构内容（例如错误的引用、内容和陈述）与正确的信息结合起来，并以有说服力和自信的方式呈现。在接下来的课程中，我们将处理所有这些限制，看看我们可以采取什么措施来减轻它们。

## 知识检查

大型语言模型的哪些说法是正确的？

1. 你每次得到完全相同的响应。
2. 它做事情完美，非常擅长添加数字，生成工作代码等。
3. 响应可能会有所不同，即使使用相同的提示。它也非常擅长为你提供某些东西的第一版，无论是文本还是代码。但是你需要逐步改进结果。

A：3，LLM是非确定性的，响应是不同的，但是你可以通过温度设置控制它的差异。你也不应该期望它做事情完美，它的作用是为你做繁重的工作，这通常意味着你会得到一次不错的尝试，需要逐步改进。
